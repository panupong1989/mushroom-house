# 01 — ภาพรวมระบบ

## เป้าหมาย
คุมสภาพอากาศโรงเพาะเห็ดฟางให้ "เห็ดออกได้จริง" โดยคุม 3 แกนพร้อมกัน โดยมี **อุณหภูมิเป็นตัวแปรหลัก**

| แกน | คุมด้วย | เป้าหมาย |
|-----|---------|----------|
| อุณหภูมิ (master) | ฮีทเตอร์ (เพิ่ม) · พัดลมดูด (ลด) | ออกดอก 28–32°C · ห้ามต่ำกว่า 25° |
| ความชื้น (รอง) | พ่นหมอก burst | 85–90% RH |
| อากาศ + แสง | พัดลมดูด (ไล่ CO₂) · หลอดไฟ | ระบายเป็นรอบ + แสงกระตุ้นออกดอก |

## เฟสการเพาะ (control profile)
- **SPAWN_RUN (เดินเส้นใย):** เป้าอุณหภูมิ "ในกอง" 32–35°C · เฝ้าระวังกองร้อนเกิน 40° (เชื้อตาย) · ยังไม่ต้องออกดอก
- **FRUITING (ออกดอก):** อากาศ 28–32°C · RH 85–90% · ระบายอากาศ+แสงเป็นรอบ

## สถาปัตยกรรม (edge-autonomous)
```
[เซนเซอร์ RS485 + DS18B20] --Modbus/1-wire--> [ESP32 ในตู้คอนโทรล]
      ESP32 รัน control FSM เอง (ทำงานต่อได้แม้เน็ตหลุด)
      |  publish telemetry / รับ command
      v  (WiFi/MQTT)
[Backend Node/TS] <--> [PostgreSQL]   <-->  [มือถือ/แดชบอร์ด]
   เก็บ readings/events, ตั้งค่า setpoint, สั่ง manual, แจ้งเตือน
```
**สำคัญ:** การตัดสินใจ real-time อยู่ที่ ESP32 (edge) — backend ไม่ใช่ authority ของ control loop
ถ้าเน็ตหลุด โรงต้องยังคุมตัวเองได้ backend แค่ sync ค่า config ลงไปเก็บใน ESP32 (NVS)
